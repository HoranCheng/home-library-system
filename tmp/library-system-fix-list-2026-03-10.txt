毛毛书房 修改清单（可执行版本）
日期：2026-03-10

说明：
这个版本按优先级整理，适合直接拿来开工、拆 issue、逐项打勾。

==================================================
P0：立刻修（公开扩散前必须处理）
==================================================

[ ] 1. 修 Worker 的 Origin 校验漏洞
目标：禁止 `startsWith()` 造成的伪造域名绕过
要做：
- 将 origin 校验改为“精确匹配”
- 本地开发 origin 也改成完整白名单（如 localhost:4173）
- Origin 为空时默认拒绝
验收：
- `https://horancheng.github.io.evil.com` 不再被允许
- 合法 origin 正常访问

[ ] 2. 给 Worker 加防滥用保护
目标：不要让 Google Books 代理被当公共代理打爆
要做：
- Cloudflare Rate Limiting / WAF
- 参数白名单
- 强制 `maxResults <= 5`
- 必要时限制 `startIndex`
- 添加缓存，减少打上游 API 次数
验收：
- 乱传参数不会直接透传
- 超大查询被裁剪
- 常见请求命中缓存

[ ] 3. 修 GitHub Pages / PWA 部署不完整问题
目标：让线上真正成为完整 PWA
要做：
- 确保以下文件被发布：
  - manifest.json
  - sw.js
  - icon-192.svg
  - icon-512.svg
- 检查 workflow，避免只复制 index.html + vendor
- 统一静态资源复制方式
验收：
- 线上这几个地址全部 200
- iPhone / Safari 可识别安装信息
- Service Worker 正常注册

[ ] 4. 修前端提示框 XSS 注入风险
目标：避免导入恶意数据后把 HTML 直接插进页面
要做：
- `makeNotice()` 内统一 escape
- 或所有动态文本调用前全部 `escapeHtml()`
- 区分纯文本 notice 与富文本 notice
验收：
- 恶意 title / error message 不会被解释成 HTML

==================================================
P1：很快修（1~2 周内）
==================================================

[ ] 5. 收紧 Service Worker 缓存策略
目标：避免缓存所有 GET 导致不可控
要做：
- 只缓存站内静态资源
- 外部 API 请求不进同一缓存策略
- 明确 cache version
验收：
- 资源升级后行为可控
- 外部资源不乱缓存

[ ] 6. 增加 CSP（Content Security Policy）
目标：缩小未来 XSS 成功后的伤害面
要做：
- 先加一个务实版本 CSP
- 放通你当前必要的 API、字体和图片域名
验收：
- 页面功能不坏
- 控制台无严重 CSP 报错

[ ] 7. 给外部请求加超时控制
目标：避免某个第三方接口慢就拖垮整次录入
要做：
- Google Books / Open Library / Bookcover / Crossref 加 AbortController
- 分级处理：拿不到分类 ≠ 整体失败
验收：
- 接口慢时仍能尽快返回主要字段

[ ] 8. 加 ISBN -> metadata 缓存
目标：提高重复查询速度，减少外部依赖抖动
要做：
- 查询成功后缓存结构化结果
- 给缓存设置版本和 TTL
验收：
- 同一本书重复录入明显更快

[ ] 9. 强化 JSON 导入校验
目标：防止脏数据、超长数据、异常结构拖垮 UI
要做：
- 校验字段类型
- 限制长度（title/author/notes/description 等）
- 限制 URL 协议
- 非法字段丢弃
验收：
- 恶意 / 极端 JSON 不会导致界面异常或注入

[ ] 10. 清理部署结构，统一 source of truth
目标：避免 preview / docs / root docs 多份产物混乱
要做：
- 明确唯一发布源目录
- 构建脚本统一产出最终发布目录
- 避免手动复制单文件
验收：
- 本地、仓库、线上三者一致

==================================================
P2：接下来做（2~4 周）
==================================================

[ ] 11. 把大单文件拆模块
目标：降低维护成本和回归风险
建议拆分：
- storage.js
- metadata.js
- scanner.js
- ui-render.js
- detail-modal.js
- styles.css
验收：
- 主文件明显变小
- 逻辑边界清晰

[ ] 12. 补浏览器 / 真机 smoke 测试
目标：不要再出现“本地正常、线上 404”这种问题
建议覆盖：
- 首页打开
- manifest / sw / icons 存在
- 导入 / 导出正常
- 搜索 / 详情弹窗正常
- 扫码入口至少能跑通 mock 流程
验收：
- 发版前能快速跑一遍

[ ] 13. 自托管字体或补隐私说明
目标：减少对 Google Fonts 的隐私暴露
要做：
- 自托管字体，或
- 在将来公开前补隐私说明

[ ] 14. 评估封面代理 / 缓存方案
目标：减少第三方直接感知用户访问行为
要做：
- 封面走自己代理或缓存
- 或明确写进隐私说明

[ ] 15. 数据安全增强
目标：降低 localStorage 单点脆弱性
要做：
- 备份提醒
- 更顺手的导出恢复
- 为未来 SQLite / 云同步留迁移路径
