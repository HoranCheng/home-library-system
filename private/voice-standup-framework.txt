一、产品定义
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

产品名称：早会 / Morning Standup（暂定）
一句话描述：开车时用语音跟 AI 开早会，讨论完自动把指令发给团队。
目标用户：你（产品负责人），管理多个产品的团队工作。

核心场景：
  早上上车 → 打开 app → AI 读早报 → 你语音讨论 → 说"结束" 
  → AI 自动整理指导意见 → 发到 Discord 对应频道 → 团队收到执行

三、核心模块
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

模块 1：早报生成与推送
  - OpenClaw 每天 7:00 AEDT 自动运行
  - 读取项目状态（TASKS.md、git log、待决事项）
  - 生成结构化早报
  - 通过 Worker API 写入 KV 存储
  - 同时发到 Discord 频道（文字版）

模块 2：语音识别（STT）
  - 技术：Web Speech API（浏览器原生，免费）
  - 语言：中文（zh-CN）
  - 模式：连续识别（continuous: true）
  - 降噪：浏览器自带（车内场景够用）
  - 备选：如果识别率不够，可换 Whisper API（$0.006/分钟）

模块 3：AI 对话
  - 模型：Claude Sonnet 4（通过你的 Max 订阅）
  - System prompt 预设：
    "你是产品负责人的早会助手。你已经有今天的产品早报。
     帮助他理解团队进展，回答他的问题，记录他的决策意见。
     用中文。简洁。不废话。"
  - 上下文：早报全文 + 对话历史
  - 支持多轮对话（完整 message history）

模块 4：语音合成（TTS）
  - 方案 A：浏览器内置 speechSynthesis（免费，声音一般）
  - 方案 B：ElevenLabs API（自然流畅，有免费额度 10k 字符/月）
  - 方案 C：OpenAI TTS API（$15/1M 字符，声音好）
  - 建议：先用方案 A 验证流程，不好听再换 B

模块 5：会议整理与发布
  - 用户说"结束会议"或点按钮
  - 把整段对话发给 Claude，prompt：
    "根据以上对话，整理出产品负责人的指导意见。
     格式参考之前的'老板指导意见汇总'。
     只输出整理后的文稿，不要多余解释。"
  - 整理好的文稿通过 Worker → Discord Webhook 发到频道
  - 用户确认后发送（或自动发送，可配置）

  [
    {
      "id": "receipt-renamer",
      "name": "小票助手",
      "discordChannel": "1480650111129354312",
      "webhookUrl": "https://discord.com/api/webhooks/xxx",
      "standupSource": "openclaw-cron"
    },
    {
      "id": "project-b",
      "name": "项目 B",
      "discordChannel": "...",
      "webhookUrl": "...",
      "standupSource": "..."
    }
  ]

开始会议时可以选产品：
  "今天要聊哪个产品？"
  → 小票助手（有新早报）
  → 项目 B（无更新）

或者一次会议聊多个产品，结束时分别发到各自的频道。

六、技术栈
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

前端：
  - React + Vite（和 receipt-renamer 一样的技术栈）
  - PWA（可加到主屏幕，全屏运行）
  - Web Speech API（语音识别）
  - SpeechSynthesis API 或 ElevenLabs（语音合成）
  - 部署：GitHub Pages

后端：
  - Cloudflare Worker（可复用 receipt-proxy 的基础设施）
  - Cloudflare KV（存储早报内容）
  - Claude API（通过 Max 订阅 token）
  - Discord Webhook（发送消息）

早报生成：
  - OpenClaw cron job（每天 7:00 AEDT）
  - 调用 Worker API 写入 KV

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
七、安全考虑
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

1. Worker API 认证
   - 简单 Bearer token（你一个人用，不需要复杂的 OAuth）
   - Token 存在 PWA 的 localStorage 里
   - Worker 校验 token 后才允许调用 Claude / Discord

2. Claude API key
   - 存在 Worker 的环境变量里，前端看不到
   - 前端只跟 Worker 通信，不直接调 Claude

3. Discord Webhook URL
   - 存在 Worker 环境变量里
   - 前端通过 Worker 中转发送，不暴露 webhook URL

4. 语音数据
   - Web Speech API 在设备端处理（iOS Safari 发到 Apple 服务器识别）
   - 对话文本发到 Claude API（Anthropic 的隐私政策：API 数据不用于训练）
   - 不存储录音，只存文本

八、开发计划
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Phase 1：最小可用版本（2-3 天）
  □ Worker 端：/api/chat、/api/standup、/api/publish
  □ 前端：单页 PWA，语音识别 + 对话 + TTS
  □ 早报：手动写入 KV（先不做自动生成）
  □ 发送：整理后一键发到 Discord
  □ 部署到 GitHub Pages

Phase 2：自动化（1 天）
  □ OpenClaw cron 每天 7 点生成早报并写入 KV
  □ 早报自动推送通知

Phase 3：体验优化（持续）
  □ 升级 TTS（ElevenLabs 或 OpenAI）
  □ 多产品支持
  □ 会议历史记录
  □ CarPlay 适配（如果 PWA 支持的话）
  □ 离线缓存（早报预加载）

  Claude API：走 Max 订阅额度           $0 额外
  Web Speech API：浏览器免费              $0
  浏览器 TTS：免费                        $0
  Cloudflare Worker：免费额度内            $0
  Cloudflare KV：免费额度内               $0
  GitHub Pages：免费                      $0
  ─────────────────────────────────────
  合计：$0 / 月（在 Max 订阅范围内）

  如果升级 ElevenLabs TTS：
  免费额度 10,000 字符/月 ≈ 每天 ~300 字
  超出后 $5/月起

  如果升级 OpenAI TTS：
  约 $0.03/次早会（1000 字）
  ≈ $0.90/月
